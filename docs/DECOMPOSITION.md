## Ядро и авторизация

### Зона ответственности

- Модуль **marketplace-api**
- Контракты (DTO, форматы ответов), используемые другими сервисами и Controller
- Качество и стабильность auth-потоков
- Тесты

### Задачи

1. **Пользователи и роли**
   - Поддержка сущностей `User`, `Role` (USER, SELLER)
   - Регистрация пользователя и продавца (`/api/user/register`, `/api/user/register/seller`)
   - Валидация входных данных, обработка ошибок (дубликаты, невалидные данные)

2. **Авторизация и JWT**
   - Логин (`/api/user/login`), выдача JWT
   - Эндпоинты для других сервисов: получение `userId` по токену, получение роли (`/api/user/userid`, `/api/user/get_role`, `/api/user/load_user_by_username`)
   - Настройка `SecurityConfig`, политики доступа к эндпоинтам
   - Безопасное хранение секретов (JWT secret и т.п.), использование переменных окружения

3. **Контракты API**
   - Поддержка и версионирование DTO: `UserDTO`, `Role` и др.
   - Документирование API (OpenAPI/Swagger) для marketplace-api
   - Согласование форматов с командой бэкенда 2 (особенно для вызовов из Controller)

4. **Инфраструктура и качество**
   - Подключение к БД (PostgreSQL), миграции схемы пользователей
   - Unit- и интеграционные тесты для сервиса и контроллеров
   - Логирование и мониторинг критичных операций (логин, регистрация)

## Бизнес-сервисы и BFF

### Зона ответственности

- Модуль **Controller** (BFF/шлюз)
- Модули **serviceForProduct**, **serviceForCast**, **ServiceForPay**
- Интеграции: Feign-клиенты к marketplace-api и между сервисами, Kafka

### Задачи

1. **Controller (BFF)**
   - Проксирование запросов к пользователям (регистрация, логин) через Feign к marketplace-api
   - Маршрутизация запросов к товарам и корзине: `/addproduct`, `/main`, `/main/{word}`, `/add_product_to_cart/{name}`, `/display/cast`
   - Сценарий покупки: `/buy_product/{productId}` — получение userId из UserClient, формирование `BuyProductDTO`, отправка в Kafka
   - JWT: проброс заголовка `Authorization` в downstream-сервисы, при необходимости — валидация/фильтр в Controller
   - Единый формат ошибок и кодов ответов для фронтенда

2. **serviceForProduct**
   - CRUD товаров, поиск по слову, получение по id/имени
   - Проверка роли SELLER при добавлении товара (через UserClient к marketplace-api)
   - Валидация, обработка ошибок (например, `NotFoundExeption`), маппинг в DTO

3. **serviceForCast**
   - Добавление товара в корзину по имени, получение содержимого корзины по пользователю (по токену)
   - Интеграция с UserClient и ProductClient для проверки пользователя и данных о товаре
   - Хранение состояния корзины (БД или кэш — по текущей реализации)

4. **ServiceForPay**
   - Обработка сообщений Kafka о покупке (`BuyProductDTO`: productId, userId)
   - Логика списания/учёта оплаты, обновление остатков (при необходимости — вызов serviceForProduct или внутренняя логика)
   - Обработка ошибок и повторные попытки (retry), идемпотентность при дублях сообщений

5. **Интеграции и инфраструктура**
   - Настройка Feign-клиентов (UserClient, ProductClient, CartClient) и их URL из конфигурации
   - Настройка Kafka: топик для покупок, consumer в ServiceForPay, при необходимости producer в Controller
   - Docker Compose / конфигурация окружений для всех сервисов
   - Тесты: интеграционные тесты Controller с моками сервисов, тесты сервисов с тестовыми БД/Kafka

## Фронт

### Зона ответственности

- Веб-фронт на реакте общающийся только с **Controller**
- UI/UX: регистрация, вход, каталог, корзина, оформление покупки
- Хранение и передача JWT

### Задачи

1. **База проекта**
   - Инициализация фронтенд-проекта (React) в репозитории (например, папка `frontend/`)
   - Публичные страницы (каталог, логин, регистрация) и защищённые (корзина, добавление товара для продавца)
   - Базовая тема, адаптивная вёрстка

2. **Авторизация**
   - Страницы: регистрация пользователя, регистрация продавца, вход
   - Запросы к Controller: `POST /register`, `POST /register_seller`, `POST /login`
   - Сохранение JWT
   - Передача токена в заголовке `Authorization` для защищённых запросов
   - Редирект на логин при 401, обработка ошибок (неверный пароль, занятый логин)

3. **Каталог**
   - Главная: список товаров `GET /main`, поиск `GET /main/{word}`
   - Карточки товаров: название, описание, рейтинг, остаток, кнопка «В корзину» / «Купить»
   - Для роли продавца: форма добавления товара `POST /addproduct` с полями из ProductDTO

4. **Корзина**
   - Страница корзины: запрос `GET /display/cast`, отображение списка товаров
   - Добавление в корзину: `POST /add_product_to_cart/{name}` (имя товара)
   - Кнопка «Оформить покупку» для выбранных позиций

5. **Покупка**
   - После выбора товара(ов) в корзине — вызов `POST /buy_product/{productId}` (при необходимости — цикл по выбранным productId)
   - Отображение результата («Оплата прошла» / сообщение об ошибке)
   - Обновление корзины после успешной оплаты (повторный запрос корзины или инвалидация состояния)
